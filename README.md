# Автотесты API для сервиса API Course

Данный проект реализует автоматизированные тесты для тестового сервера API-курса. Тесты написаны с использованием 
Python, Pytest, Allure, Pydantic, Faker и HTTPX. Исходный код тестируемого приложения доступен на GitHub:
https://github.com/Nikita-Filonov/qa-automation-engineer-api-course.

## Обзор проекта

Цель проекта — автоматизировать тестирование сервера API-курса с фокусом на тестирование REST API. Автоматизированные 
тесты проверяют различные функции приложения для обеспечения его стабильности и корректности работы.

Проект специально разработан для API-автотестов и включает такие лучшие практики, как:

- API-клиенты для структурированного взаимодействия с эндпоинтами;
- Pytest-фикстуры для переиспользуемых и поддерживаемых тестовых настроек;
- Pydantic-модели для строгой валидации данных;
- Валидация схем для проверки соответствия API-контракту;
- Генерация тестовых данных для имитации реальных сценариев;
- И другие продвинутые техники для повышения эффективности и надежности тестов.

## Начало работы

### Клонирование репозитория

Для начала работы нужно клонировать репозиторий с GitHub

```bash
git clone https://github.com/leavecoy/autotests-api.git
cd your-repo-name
```

### Cоздание виртуального окружения

Рекомендуется использовать виртуальное окружение для управления зависимостями проекта. Следуйте инструкциям для вашей 
операционной системы:

#### Linux / MacOS

```bash
python3 -m venv venv
source venv/bin/activate
```

#### Windows

```bash
python -m venv venv
venv\Scripts\activate
```

### Установка зависимостей

После активации виртуального окружения установите зависимости проекта из requirements.txt:

```bash
pip install -r requirements.txt
```

### Запуск автотестов с генерацией Allure-отчета

Для запуска тестов и генерации отчета используйте данную команду:

```bash
pytest -m "regression" --alluredir=./allure-results
```

Эта команда выполнит все тесты в проекте и отобразит результаты в терминале, а также сохранит необходимые для генерации
Allure-отчета файлы в корневой директории проекта, в папке allure-results

### Просмотр Allure отчёта

После полного выполнения тестов можно сгенерировать Allure-отчет с помощью следующих команд:

```bash
allure serve allure-results
```

Эта команда откроет Allure-отчет в браузере по умолчанию

```bash
allure generate ./allure-results -o ./allure-report --single-file
```


Эта команда сгенерирует Allure-отчет одним файлом в папке allure-report в корневой директории проекта

### Просмотр отчета о покрытии эндпоинтов с помощью Swagger Coverage Tool


```bash
swagger-coverage-tool save-report
```

Эта команда сгенерирует отчет о покрытии эндпоинтов автотестами.

### Структура проекта

- .github: Папка с настройками пайплайнов для GitHub Pages.
- clients: Пакет с API-клиентами и описанием используемых Pydantic-моделей.
- fixtures: Пакет с используемыми в тестах фикстурами.
- testdata: Тестовые файлы, необходимые для выполнения некоторых тестов.
- test: Пакет с автотестами.
- tools: Пакет с вспомогательными утилитами, такими как описание структуры Allure-отчета, настройкой assertions и т.д.
- .env: Переменные окружения.
- config: Файл, отвечающий за инициализацию настроек проекта при запуске.
- conftest: Файл для подключения фикстур в виде плагинов.
- pytest.ini: Файл с конфигурациями настроек запуска Pytest.
- requirements.txt: Файл с необходимыми для работы автотестов библиотеками.
